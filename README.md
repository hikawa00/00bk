# 个人博客（纯前端版）

一个简单但比较精致的个人博客模版，只有 HTML / CSS / JavaScript，适合本地浏览或部署到任意静态网站托管平台（例如 GitHub Pages、Vercel、Netlify 等）。

> 你可以直接在浏览器中打开 `index.html` 使用，不依赖任何后端服务。

---

## 功能简介

- **文章列表**
  - 按时间倒序展示文章
  - 显示标题、日期、分类、摘要
- **文章详情**
  - 单页应用（SPA）效果，无需整页刷新
  - 通过地址栏 `#post=...` 路由直接打开某篇文章
- **搜索**
  - 支持按标题、摘要、正文内容搜索（模糊匹配）
- **分类筛选**
  - 固定五个分类：学习、探索、欣赏、创作、积累
  - 左侧「分类」区域点击某一类，只看该类文章
- **自适应布局**
  - 在 PC、Pad、手机上都有良好的阅读体验
- **自我介绍页**
  - 顶部导航可切换到「关于我」页
  - 支持头像（圆形块）+ 介绍文字
- **本地写作后台页**
  - 通过 `editor.html` 打开
  - 填写表单自动生成一段可直接复制到 `main.js` 的文章对象代码

## 快速开始

### 1. 直接打开

1. 在资源管理器中打开本项目所在文件夹。
2. 双击 `index.html`，用浏览器打开即可。

> 这种方式最简单，不需要任何命令行操作。

### 2. 使用本地静态服务器（推荐）

如果你希望 URL 更干净、体验更接近线上部署，可以用 `npm` 启动一个本地静态服务器。

#### 安装依赖

在项目根目录（包含 `package.json` 的这个目录）打开终端，运行：

```bash
npm install
```

#### 启动开发服务器

```bash
npm run dev
```

然后在浏览器访问终端里提示的地址（通常是 `http://localhost:4173` 或类似端口）。

## 如何写文章 / 修改内容

所有文章数据都在 `main.js` 顶部的 `posts` 数组中，你可以按示例格式添加、修改或删除文章。

### 文章数据结构

每一篇文章是一个对象，例如：

```js
{
  id: "hello-blog",              // 唯一 ID，用于路由，建议使用英文或拼音
  title: "你好，博客世界",        // 标题
  date: "2026-03-01",            // 日期（任意可解析的日期格式，推荐 YYYY-MM-DD）
  tags: ["学习"],                // 分类，这里只能是：学习 / 探索 / 欣赏 / 创作 / 积累
  excerpt: "这一段会显示在列表中，作为摘要。",
  content: `
    <p>这里是正文内容，支持原生 HTML。</p>
    <h2>小标题示例</h2>
    <ul>
      <li>你可以使用 <code>&lt;p&gt;</code>、<code>&lt;h2&gt;</code> 等标签</li>
    </ul>
  `,
}
```

> `content` 字段是一个模板字符串（用反引号 `\`` 包起来），里面可以写原生 HTML。  
> 如果你熟悉 Markdown，也可以自己引入第三方库来转换成 HTML。

### 添加新文章

1. 打开 `main.js`。
2. 找到开头的 `const posts = [ ... ];`。
3. 按照上面的结构，在数组末尾添加一个对象（记得用逗号分隔）。
4. 保存文件，刷新浏览器即可看到新文章。

## 主题与样式

- 全局样式在 `styles.css` 中，可以：
  - 修改颜色变量（文件顶部 `:root` 部分，目前是固定的浅色紫蓝风格）
  - 调整圆角等
- 页面结构在 `index.html` 中：
  - 左边是侧边栏（搜索、分类、简要关于）
  - 右边是内容区域（自我介绍 / 文章列表 / 详情切换）

- 自我介绍页在 `index.html` 中的 `introView` 区块：
  - `intro-avatar-circle`：默认显示 “ME” 的圆形头像块，你可以改成自己的姓名缩写
  - 也可以把这一块换成 `<img>` 标签来使用真实头像图片

## 写作后台页面（editor.html）的使用方法

1. 在资源管理器中，双击打开 `editor.html`（和 `index.html` 一样，直接用浏览器打开即可）。
2. 在左侧表单中填写：
   - **标题**：文章标题
   - **日期**：默认是今天，可以自行修改
   - **ID**：可以留空，会根据标题自动生成英文/拼音风格的 ID；也可以自己手动填写
   - **分类**：从「学习 / 探索 / 欣赏 / 创作 / 积累」中选择一个
   - **摘要**：文章列表中展示的一两句话
   - **正文**：支持原生 HTML 标签，例如：

     ```html
     <p>这一段是正文内容。</p>
     <h2>小标题</h2>
     <ul>
       <li>要点 1</li>
       <li>要点 2</li>
     </ul>
     ```

3. 点击底部「生成代码片段」按钮。
4. 右侧会出现一段形如：

   ```js
   {
     id: "your-id",
     title: "你的标题",
     date: "2026-03-01",
     tags: ["学习"],
     excerpt: "这里是一两句话的摘要。",
     content: `
       <p>这里是正文内容。</p>
     `,
   },
   ```

5. 复制这段代码：
   - 打开 `main.js`
   - 找到顶部的 `const posts = [ ... ];`
   - 在数组末尾粘贴生成的对象（注意保留每一项之间的逗号）
6. 保存 `main.js`，刷新 `index.html` 页面，就能在博客中看到新文章。

修改前可以先备份一份，以便不满意时恢复。

## package.json 说明

项目根目录下的 `package.json` 主要用于：

- 管理依赖（目前只用来启动一个简易静态服务器）
- 提供 `npm run dev` 这样的快捷命令

你可以自由添加自己需要的工具，例如打包工具、CSS 预处理器等。

## 部署到线上（简单思路）

因为这是一个纯静态网站，你只需要把以下文件上传到任意静态托管服务即可：

- `index.html`
- `styles.css`
- `main.js`

例如：

- GitHub Pages
- Vercel
- Netlify
- 或者任何支持静态文件的服务器 / 对象存储
